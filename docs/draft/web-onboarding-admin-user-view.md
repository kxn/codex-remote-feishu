## 2. 用户如何理解这两个页面

### 2.1 setup

`setup` 是一次性的安装向导。

它只负责：

- 把这台机器带到“已经可以开始使用”的状态

### 2.2 admin

`admin` 是后续的日常管理页。

它负责：

- 看机器人状态
- 新增一个机器人
- 看机器集成状态
- 管理 Claude 配置
- 清理存储占用

用户不应该再感受到：

- `setup` 是一个长期使用的大控制台
- `admin` 是一个共享 onboarding 工作台

## 3. 用户打开 setup 后会看到什么

setup 继续保持 `v1.7.0` 的向导外观：

- 左侧步骤栏
- 右侧当前步骤
- 页面顶部只有标题和反馈

用户会按顺序经过这些步骤：

1. `环境检查`
2. `飞书连接`
3. `权限检查`
4. `事件订阅`
5. `回调配置`
6. `菜单确认`
7. `自动启动`
8. `VS Code 集成`
9. `完成`

这里最重要的约束是：

- 页面一次只讲一个步骤
- `事件订阅` 和 `回调配置` 仍然留在 setup
- 页面不再追加全局总览、推荐处理、剩余建议项之类的文字

## 4. 用户打开 admin 后会看到什么

admin 继续保持 `v1.7.0` 的默认主结构，并只新增 `Claude 配置` 一块。

当前默认顺序是：

1. `机器人管理`
2. `系统集成`
3. `Claude 配置`
4. `存储管理`

### 4.1 机器人管理

- 左侧列出现有机器人和 `新增机器人`
- 右侧显示当前机器人的状态页
- 新增机器人时，右侧出现和 setup 很像的连接流程

### 4.2 系统集成

- 只保留 `自动运行设置`
- 只保留 `VS Code 集成`

### 4.3 Claude 配置

- 当前直接在 admin 里管理
- 这是在 `v1.7.0` 基础上新增的唯一默认主界面区块

### 4.4 存储管理

- `预览文件`
- `图片暂存`
- `日志文件`

## 5. 用户现在不该再看到什么

以下内容不应再默认进入 setup/admin 主界面：

- 共享 onboarding workflow 叙事
- “当前推荐处理”
- “剩余建议项”
- 运行概览
- 工作实例
- 技术详情大分区
- 大段解释性文案

## 6. 当前结论

当前接受的页面方向很简单：

- `setup` 回到 `v1.7.0` 的单步向导
- `admin` 回到 `v1.7.0` 的管理页结构
- 在 `admin` 上只额外加入 `Claude 配置`

除此之外，不再追加新的默认主界面结构。
