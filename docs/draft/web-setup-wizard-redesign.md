# Web Setup 向导流程重设计

> Type: `draft`
> Updated: `2026-04-29`
> Summary: 当前 setup 页面以 `v1.7.0` 为基线，保留左侧步骤栏和右侧单步正文；权限检查支持强制跳过，事件与回调继续留在 setup 内完成。

## 2. 用户可见合同

- 最终用户：首次完成网页安装配置的普通用户
- 当前任务：把当前机器带到“可以开始使用”的状态
- 允许展示：当前步骤、当前步骤结果、当前步骤所需后台入口、当前步骤的继续按钮
- 不允许展示：额外总览、共享控制台叙事、和当前步骤无关的大段解释
- 反馈槽位：页顶全局通知、当前步骤正文内的状态条和按钮区

## 3. 页面结构

setup 继续保持 `v1.7.0` 的基础结构：

- 左侧步骤栏
- 右侧当前步骤正文
- 页面顶部只保留标题和简短反馈

## 4. 固定步骤

当前步骤顺序固定为：

1. `环境检查`
2. `飞书连接`
3. `权限检查`
4. `事件订阅`
5. `回调配置`
6. `菜单确认`
7. `自动启动`
8. `VS Code 集成`
9. `完成`

## 5. 权限检查规则

权限检查现在采用下面这条正式规则：

- 用户可以先补齐权限，再继续
- 用户也可以 `强制跳过这一步`
- 跳过后直接继续后面的 setup 步骤
- 后续仍可回到权限检查重新检查

这条规则属于当前有效产品结论，不再视为草案或例外。

## 6. 当前结论

- setup 不改成新的总控台页面
- `事件订阅` 和 `回调配置` 继续放在 setup 内
- 新流程变化优先消化在现有步骤内部
- 如果以后要改默认页面结构，必须先改文档和 mock，再改产品页面
