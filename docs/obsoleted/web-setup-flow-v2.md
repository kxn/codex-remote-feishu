# Web Setup / Web Admin 改造方案与技术调研（V2 合并版）

> Type: `obsoleted`
> Updated: `2026-07-17`
> Summary: 历史 Web Setup/Admin V2 方案，仍描述旧权限/事件/回调分步流程；当前飞书接入 SSOT 已转为自动配置路径。
> Superseded By: `docs/implemented/web-admin-ui-redesign.md`

> 注意：本文是历史归档，不再作为当前实现依据。文中的“当前”均指归档前的旧基线。

## 1. 当前页面基线

- setup：继续使用 `v1.7.0` 向导结构
- admin：继续使用 `v1.7.0` 管理页结构
- admin 默认新增：`Claude 配置`、`Codex Provider`

## 2. 当前流程基线

- `事件订阅` 和 `回调配置` 继续留在 setup
- `新增机器人` 的连接和权限处理继续留在 admin 的机器人管理区
- 权限检查允许 `强制跳过这一步`

## 2.1 当前交互细节基线

- setup 页面标题与浏览器标题使用后端真实版本号，例如 `Codex Remote Feishu v1.7.0 安装程序`
- admin 页面标题与浏览器标题同样使用后端真实版本号，不再保留前端占位版本
- setup 的 `飞书连接` 与 admin 的 `新增机器人` 使用同一套连接交互：
  - `扫码创建` 和 `手动输入`
  - 扫码默认自动开始，并按 2 秒轮询
  - 手动输入的必填项是 `App ID` 和 `App Secret`，`机器人名称` 保持可选
- setup 内的权限、事件订阅、回调配置、菜单配置都使用后端返回的飞书后台真实链接；前端不再自己拼默认应用首页
- 安装期的事件订阅测试与回调测试都走“当前机器人绑定的 web test recipient”，不再依赖 recent-surface fallback
- 安装期测试状态只在当前阶段短时有效；超时、测试通过、阶段切换或配置变化后都应立即失效

## 3. 当前使用方式

以后继续改相关页面时，按下面顺序判断：

1. 这次改动是否还在 `v1.7.0` 页面结构内？
2. 如果不在，是否真的需要改默认主界面结构？
3. 如果需要，是否已经先同步文档和 mock？

任一步答案为否，都不应直接扩页面。
