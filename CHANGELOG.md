# Changelog

## v1.4.0 (待发布)

覆盖范围：自 `v1.3.0` 以来的用户可感知变化。

### 安装和开始使用更顺了

- 现在可以直接按 `production / beta / alpha` 三条 track 安装，想尝鲜或回到正式版都更直接。
- macOS 安装链路补了兼容性修复，在线安装脚本在空参数场景下也不会再意外报错。
- WebSetup 的引导文案、回调说明和 VS Code 接入说明重新整理过，首次配置时更容易按顺序走通。

### 飞书里更像“继续当前工作”，而不是“重新找上下文”

- `/help`、`menu` 和 `/menu` 现在可以直接把当前可用命令和快捷入口展示出来，不用再靠记忆。
- 最终回复会直接回在你触发它的那条消息下面，群聊里更容易看懂上下文是谁发起的。
- 系统卡片里对代码片段和行内 token 的渲染更清楚，最终卡片标题也会带一点原始提问预览。
- 如果一条文字还在排队，你可以给这条消息点个赞，把它升级成对当前正在执行回复的跟进或纠偏。

### 旧卡片和旧动作现在会更明确地“失效”，减少误触

- daemon 重启后，属于上一个生命周期的旧消息、旧菜单动作和旧卡片会被统一识别并明确提示，不再悄悄落到当前流程里。
- 旧的 `/newinstance`、`/killinstance` 和历史恢复卡片已经退出主流程；看到迁移提示时，直接改用 `/use`、`/useall` 或 `/detach` 即可。
- 旧卡片如果已经过期，会直接告诉你重新发哪条命令拿新卡片，而不是继续沿用一张已经失真的按钮卡。

### 恢复和稳定性更好了

- daemon 重启后的后台恢复流程更稳：恢复线索会持久化，恢复会延后到合适时机再进行，不会一上来就误判失败。
- `/detach` 现在不仅能断开接管，也能顺手取消当前还没完成的恢复流程，逃生路径更明确。
- 本地链路波动时，正在执行的 turn 更不容易被中途打断；队列余量和停机通知也都更完整。
- 管理页里不再把纯后台恢复实例暴露成普通可接管对象，日常视图更聚焦。

### 升级和回滚基础补齐了

- 安装状态现在会记录更完整的版本与来源信息，为后续升级检查、候选调试和自动回滚打下基础。
- 已经补上 helper 驱动的升级回滚流程，切换版本时对正在运行的 wrapper 和 daemon 处理更稳。
